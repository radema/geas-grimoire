Engineering the Intent-Centric Repository: A Feasibility Study of Spec-Driven Development and Agentic OrchestrationThe contemporary software engineering landscape is currently traversing a structural inflection point, shifting from a paradigm of manual code authorship toward a model of autonomous intent orchestration. This transition, frequently characterized as the transition from "Vibe Coding" to "Spec-Driven Development" (SDD), necessitates a fundamental re-evaluation of the relationship between human architects and agentic executors. In this emerging ecosystem, the primary technical artifact is no longer the source code itself, but the specification—the "Elephant"—which preserves long-term architectural intent, while the implementation—the "Goldfish"—is treated as a transient, ephemeral product of agentic labor. The feasibility of this model relies on the ability to maintain a state-aware repository where documentation and code are mathematically and semantically synchronized. This report provides an exhaustive feasibility study of the "Bolt" skill-set within the context of industry frameworks, state-of-the-art tooling, and the inherent risks of semantic drift, identifying the precise thresholds where structured specifications transition from a strategic asset to a maintenance burden.The Evolution of Spec-Driven Development and Industry BenchmarkingTo understand the positioning of the Bolt workflow, one must first locate it within the broader spectrum of industry frameworks that have emerged to manage the complexities of agentic coding. Spec-driven development is not a localized trend but a global movement toward elevating abstraction to the system level. Frameworks such as the GitHub Spec-kit, OpenSpec, and GSD (Get Stuff Done) represent various philosophies along the spectrum of specification rigor. While vibe coding offers immediate, low-latency results for isolated tasks, it lacks the durable technical artifacts required for professional engineering.The GitHub Spec-kit, for instance, establishes a four-phase guardrail: Specify, Plan, Break Down Tasks, and Implement. This methodology ensures that AI agents operate within a predefined technical stack and architectural pattern, rather than making probabilistic guesses based on vague prompts. While git diff is effective at identifying that a file has changed, it cannot inherently understand why it changed or whether that change violates the original intent.The Mechanism of Semantic DriftSemantic drift occurs when the implementation remains syntactically valid (it compiles and runs) but its business logic diverges from the specification. This is particularly dangerous in agentic workflows where the agent might "synthesize" a fix that inadvertently bypasses a security constraint or alters an API contract.Case studies on "Automated Program Repair" suggest that LLMs often struggle with "Semantic Equivalence"—the ability to reason about whether two different implementations fulfill the same intent.
